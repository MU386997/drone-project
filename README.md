Miniature Intelligent Drone Project – CropWatcher

A Crazyflie-based intelligent drone system designed for indoor greenhouse monitoring, environmental mapping, and crop health prediction using real-time sensor telemetry, autonomous flight, and machine learning.

⸻

Project Overview

CropWatcher is a graduate design project focused on using autonomous drone technology to improve greenhouse monitoring and crop management. The system combines:

* Autonomous indoor drone navigation
* Real-time environmental telemetry
* Lighthouse positioning
* Sensor data collection and correction
* Machine learning-based crop health prediction
* Interactive web-based monitoring and mission planning

The platform was developed using the Crazyflie 2.1 drone along with the Lighthouse indoor positioning system and onboard environmental sensors.

Main Components

1. Waypoint Flight System

The waypoint system allows users to create autonomous flight missions by selecting points on a 5x5 foot interactive map. The drone uses Lighthouse positioning for accurate indoor navigation.

Features include:

* Interactive GUI waypoint selection
* Relative-to-origin flight planning
* Static obstacle checking
* Operational area enforcement
* Sensor logging during missions
* Autonomous takeoff and landing

The waypoint system integrates with the same telemetry and logging framework used by the lawnmower flight system.  ￼

⸻

2. Lawnmower Flight System

The lawnmower flight script autonomously scans the greenhouse in a structured coverage pattern. This flight mode was designed for efficient environmental mapping and sensor data collection across the greenhouse area.

Functions include:

* Automated area coverage
* Continuous telemetry logging
* CSV data export
* Environmental scanning
* Real-time frontend integration

⸻

3. Front-End Dashboard

The frontend provides a centralized control and monitoring interface for the project.

Features include:

* Live telemetry display
* Mission execution controls
* Flight status monitoring
* Greenhouse mapping
* Historical CSV data loading
* Mission planning tools
* Machine learning visualization

⸻

4. API Server

The backend API server acts as the communication bridge between the drone systems and the frontend dashboard.

Capabilities include:

* Live telemetry streaming
* Flight script launching
* Mission queue management
* CSV file management
* Geofence handling
* Frontend communication
* LLM-assisted mission generation support

The API server manages telemetry state, mission execution, and frontend communication.  ￼

⸻

5. Temperature Correction Algorithm

One challenge discovered during testing was that heat generated by the drone electronics affected the onboard BMP388 temperature readings.

To improve accuracy, a correction algorithm was developed that:

* Estimates environmental ambient temperature
* Detects thermal drift
* Compensates for processor heat
* Adjusts for flight conditions and airflow
* Produces corrected environmental readings

This significantly improved environmental monitoring reliability.

⸻

6. Machine Learning System

The machine learning portion of the project uses environmental sensor data to predict crop health conditions.

The ML system:

* Processes sensor telemetry
* Generates crop health predictions
* Creates greenhouse health maps
* Uses trained neural network models
* Integrates directly with the frontend dashboard

The repository includes:

* Trained .pt model files
* Jupyter notebooks for analysis
* Prediction and visualization workflows

⸻

Lighthouse Positioning System

The Lighthouse positioning system provides accurate indoor localization for the Crazyflie drone.

Benefits include:

* Precise indoor navigation
* Stable waypoint following
* Accurate environmental mapping
* Repeatable autonomous missions

The system uses:

* Lighthouse base stations
* Crazyflie Lighthouse deck
* Kalman state estimation

⸻

Data Logging

Flight and sensor data are logged to CSV files for later analysis.

Logged data includes:

* Timestamp
* Raw temperature
* Corrected temperature
* Pressure
* Altitude
* Air density
* Battery voltage
* Lighthouse X/Y position
* Flight telemetry

The system automatically organizes flight data into date-based folders to prevent overwriting previous missions.

Running the Project

1. Install Dependencies

pip install cflib pandas matplotlib numpy torch

⸻

2. Start the API Server

python api_server_final.py

⸻

3. Open the Frontend

Open:

mainFrontEnd_final.html

or navigate to the hosted API frontend page.

⸻

4. Run Flight Scripts

Waypoint Flight

python waypoint_flight.py

Lawnmower Flight

python lawnmower_flight.py

Hover Test

python hover_test.py

Safety Notes

* Ensure Lighthouse positioning is calibrated before flight
* Keep the drone within the configured operational area
* Verify battery levels before launch
* Test missions in dry-run mode first
* Maintain clear indoor flight space
